# Документація VolunteerFleet

Технічна документація проєкту: архітектура, прийняті рішення та довідник по підсистемах.
Мета — щоб новий контриб'ютор (людина або AI-агент) міг зрозуміти **не лише як влаштована
система, а й чому вона влаштована саме так**, і почати працювати без додаткових питань.

> Мовні правила проєкту: код, ідентифікатори, назви таблиць/колонок і коміти — англійською;
> UI-тексти й документація для кінцевих користувачів — українською. Ця технічна документація
> ведеться українською як основна мова спілкування в проєкті.

## Принципи цієї документації

1. **Джерело правди — код.** Документація описує реальну реалізацію; якщо вони розходяться — правий код, а документ треба оновити.
2. **Пояснюємо «чому», а не лише «що».** Структуру коду видно з самого коду; цінність документації — у зафіксованих причинах рішень і відкинутих альтернативах.
3. **Свідомі спрощення позначені явно.** Те, що навмисно не реалізовано в межах першої версії, винесено в розділи «Поза межами» з планом на майбутнє.
4. **Оновлюється разом зі змінами.** Зміна архітектури → відповідний апдейт документа в тому ж PR.
