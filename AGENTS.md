# AGENTS.md

Orientation file for AI agents working on this project. Keep it short — add details as they become real decisions rather than assumptions.

## About the project

**VolunteerFleet** — a web platform for tracking vehicles donated to the Armed Forces of Ukraine by volunteer initiatives. Unified registry of vehicles, expenses, supporting documents, and donor-facing reports.

## Language rules

- **English** — code: identifiers (variables, functions, classes, files, NestJS modules), DB tables/columns, API keys, commit messages, code comments, technical READMEs in the repo.
- **Ukrainian** — UI texts in the app, chat communication with the user, end-user-facing documentation.
- If a code comment is genuinely needed, keep it English and minimal.

## Tech summary

- **Monorepo** with pnpm workspaces: `apps/server`, `apps/client`, `packages/shared`.
- **Backend:** NestJS + TypeScript + Drizzle ORM + PostgreSQL 16 + Zod (validation) + JWT (access + refresh httpOnly) + `@aws-sdk/client-s3` (MinIO).
- **Local PostgreSQL:** runs in a dedicated Docker container on host port `55432` (not the default `5432`). Canonical values — `POSTGRES_PORT=55432` and `DATABASE_URL=postgresql://volunteerfleet_user:change_me_in_local@localhost:55432/volunteerfleet_dev`.
- **Frontend:** React 18 + Vite + Ant Design v5 + TanStack Query + Zustand + react-router v6.
- **Files:** MinIO via docker-compose; upload + link modes; presigned download.
- **Multi-currency:** UAH (base) + USD + EUR; monthly rates from JSON; `rate` is stored on each expense.
- **API:** REST `/api/v1`, OpenAPI/Swagger autogenerated from Zod schemas.
- **Tests:** Vitest for BE + FE. Husky + lint-staged. GitHub Actions CI.
- **Run (dev):** `pnpm infra:up && pnpm db:migrate && pnpm db:seed && pnpm dev`.

## Git workflow

- Base branch: **`main`**.
- Feature branches: `feat/<short-name>`, `fix/<short-name>`, `chore/<short-name>`, etc. (Conventional Commits style).
- Always `git fetch origin` before creating a new branch.
- Before preparing a release, read `docs/deployment.md` and follow its release workflow.
- Agents do **not** push to `main` directly — open a PR.

## Conventions

- Don't invent conventions that aren't already in the repo. If something is unclear — stop and ask the user.
- Don't add features, refactors, or abstractions beyond what the task requires.
- Avoid unnecessary comments. Identifiers should be self-explanatory; comments are reserved for non-obvious *why*.
- Don't introduce dependencies without explicit confirmation.

## Tooling

- **mise** ([mise.jdx.dev](https://mise.jdx.dev/)) — recommended for managing Node/pnpm versions and auto-loading `.env`. Config: `.mise.toml`. Alternative: nvm + corepack.
- **Claude Code permissions:**
  - `.claude/settings.json` — project policy (committed, shared across agents): allows read-only Bash and safe git/pnpm commands; denies destructive operations (`rm -rf`, force push, push to `main`, publishing, `docker compose down -v`, etc.).
  - `.claude/settings.local.json` — local trust (gitignored): edit/write, adding deps, push to feature branches. DB commands (`db:migrate`, `db:seed`, drizzle-kit) are in the `ask` list and require confirmation.
